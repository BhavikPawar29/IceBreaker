# IceBreaker

IceBreaker is a community-curated conversation idea app for shy people who already know someone a little, but do not know what to ask next.

It is not built for cold pickup lines or AI-generated filler. The product is centered on human-written conversation starters, community voting, and a promotion flow that filters out weak ideas over time.

## What the product does

- Lets signed-in users browse conversation ideas from the community
- Lets users submit their own ideas to the board
- Lets users vote on candidate ideas that still need validation
- Promotes strong ideas into a read-only top-picks section
- Gives each user a profile page with the ideas they have contributed
- Supports public sharing for individual ideas and public profiles

## Product positioning

IceBreaker is meant for awkward, half-familiar social moments:

- someone you already know a bit
- someone you have met before
- someone you want to talk to more naturally

The core promise is simple: better things to say, filtered by people instead of generated by AI.

## How it works

1. Users sign in.
2. They browse candidate ideas or promoted ideas.
3. Candidate ideas can be voted on by the community.
4. Useful ideas rise into the promoted section.
5. Users can also submit their own ideas and track them from their profile.

## App structure

Current main routes:

- `/` landing page and product explanation
- `/login` sign-in and sign-up
- `/lines` candidate ideas that can still be voted on
- `/promoted` top picks that already proved useful
- `/create` submit a new idea
- `/profile` signed-in user's own contributions
- `/profile/:id` public profile page
- `/line/:id` public single-idea page

## Local development

Install dependencies:

```bash
npm install
```

Start the app:

```bash
npm run dev
```

Build for production:

```bash
npm run build
```

Run lint:

```bash
npm run lint
```

## Environment setup

Create a `.env` file from `.env.example` and fill in your Firebase web app values:

```env
VITE_FIREBASE_API_KEY=
VITE_FIREBASE_AUTH_DOMAIN=
VITE_FIREBASE_PROJECT_ID=
VITE_FIREBASE_STORAGE_BUCKET=
VITE_FIREBASE_MESSAGING_SENDER_ID=
VITE_FIREBASE_APP_ID=
```

If Firebase config is missing, the UI still renders, but authentication and Firestore syncing stay disabled.

## Deployment

This repo includes Firebase deployment support:

```bash
npm run firebase:deploy
```

That deploy script is currently configured to deploy Firebase Hosting and Firestore rules/indexes.
